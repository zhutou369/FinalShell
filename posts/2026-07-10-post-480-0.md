## 准备工作：磨刀不误砍柴工

在开始批量操作之前，充分的准备是确保流程顺利、安全的关键。

### 1. 确保 FinalShell 已就绪

首先，你需要确保你的 FinalShell 客户端已经正确安装并能稳定运行。如果你还没有安装 FinalShell，或者对安装过程有疑问，可以参考我们的详细教程 [FinalShell Windows 安装与设置](/posts/cn-finalshell-windows-install-setup/)，确保客户端环境万无一一失。

### 2. 梳理服务器列表与当前凭证

*   **明确目标服务器：** 确定需要修改密码的所有服务器 IP 地址或域名列表。
*   **收集当前登录凭证：** 你需要拥有这些服务器当前的登录用户名（通常是 `root` 或具有 sudo 权限的用户）和密码。这是执行批量修改操作的前提。
*   **新密码：** 提前想好一个安全、符合密码策略的新密码。请务必使用强密码，结合大小写字母、数字和特殊符号，长度至少12位。

### 3. 网络连通性检查

确保你的 FinalShell 客户端可以正常连接到所有目标服务器。如果某些服务器连接不上，或者存在 [FinalShell 连接超时问题](/posts/cn-finalshell-connect-timeout-fix/)，请务必先解决这些问题，否则批量操作将无法成功。

### 4. 风险提示与备份（重要！）

*   **操作风险：** 批量修改密码是高风险操作。一旦操作失误，可能导致大量服务器无法登录。
*   **提前备份：** 在进行任何批量操作前，强烈建议你为关键服务器创建快照或备份重要的配置文件和数据。虽然修改密码通常不会导致数据丢失，但以防万一。
*   **小范围测试：** 如果可能，先选择一两台非核心服务器进行小范围测试，确认流程无误后再推广到所有服务器。

## 核心策略：SSH 脚本批量执行

FinalShell 并没有一个直接的“批量修改密码”的 GUI 按钮，因为密码修改涉及到底层操作系统的 `passwd` 命令，通常需要非交互式地执行。FinalShell 的强大之处在于其“分组管理”和“批量执行脚本”功能，这正是我们实现目标的关键。

我们将利用 SSH 连接到每台服务器，并执行一条 shell 命令来修改密码。下面是 FinalShell 的核心功能界面，你可以看到它提供了丰富的管理选项。

### 步骤一：创建服务器分组并导入列表

在 FinalShell 中，首先将你的目标服务器组织成一个或多个分组，这有助于管理和批量操作。

1.  **打开 FinalShell 客户端。**
2.  **创建分组：** 在左侧的服务器列表区域，右键点击空白处或“会话”节点，选择“新建分组”。给分组起一个有意义的名字，例如“待修改密码服务器”。
3.  **导入服务器：**
    *   你可以一台一台地在分组中添加服务器，填写连接信息（IP、端口、用户名、密码）。如果你是 [FinalShell 初次连接服务器](/posts/cn-finalshell-first-server-connection/)，应该对这个过程很熟悉。
    *   更高效的方法是批量导入。如果你的服务器列表是规范的文本文件，FinalShell 支持从文件导入。右键点击分组，选择“导入会话”。根据提示选择你的服务器列表文件（通常是 CSV 或自定义格式），并配置好导入规则。

确保所有需要修改密码的服务器都已添加到这个分组中，并且它们的当前登录凭证是正确的，否则无法建立 SSH 连接执行后续操作。

### 步骤二：编写密码修改脚本

为了实现非交互式密码修改，我们将利用 Linux 系统自带的 `chpasswd` 命令。`chpasswd` 专为批量修改密码而设计，它从标准输入读取 `用户名:新密码` 格式的数据。

假设我们要将 `root` 用户的密码修改为 `MyNewSecureP@ssw0rd2026!`。

**脚本示例：**

```bash
#!/bin/bash

# 定义新密码
NEW_PASSWORD="MyNewSecureP@ssw0rd2026!"

# 定义要修改密码的用户名
USERNAME="root"

# 使用 chpasswd 命令修改密码
# 注意：此命令需要 root 权限或者 sudo 权限
echo "${USERNAME}:${NEW_PASSWORD}" | chpasswd

# 检查命令执行结果
if [ $? -eq 0 ]; then
    echo "用户 ${USERNAME} 的密码已成功修改。"
else
    echo "修改用户 ${USERNAME} 密码失败，请检查权限或日志。"
fi
```

**重要说明：**

*   **`chpasswd` 权限：** 执行 `chpasswd` 命令通常需要 `root` 权限。如果你当前登录 FinalShell 的用户不是 `root`，你需要确保该用户拥有 `sudo` 权限，并且可以免密码执行 `chpasswd`，或者在脚本中加入 `sudo`。例如：`echo "${USERNAME}:${NEW_PASSWORD}" | sudo chpasswd`。如果 `sudo` 需要密码，那么这个非交互式脚本就会卡住。最简单直接的方式就是使用 `root` 用户登录 FinalShell 来执行这个脚本。
*   **替换密码：** 请务必将 `MyNewSecureP@ssw0rd2026!` 替换为你自己设定的新密码。
*   **替换用户名：** 如果你要修改的是其他用户的密码，请将 `USERNAME="root"` 替换为对应的用户名。
*   **`passwd --stdin` 替代方案 (不推荐优先使用)：**
    有些老旧或特定配置的系统可能不方便使用 `chpasswd`，或者 `chpasswd` 命令不存在。此时，可以考虑使用 `echo "新密码" | passwd --stdin 用户名`。但请注意，**很多现代 Linux 发行版出于安全考虑，默认禁用了 `passwd --stdin` 管道输入功能**。如果你的服务器禁用此功能，上述脚本将无效。因此，`chpasswd` 是更可靠的选择。

### 步骤三：在 FinalShell 中批量执行脚本

现在，我们将利用 FinalShell 的批量执行功能，将上述脚本分发并执行到选定的服务器分组上。

1.  **选择服务器分组：** 在 FinalShell 左侧的服务器列表中，选中你之前创建的服务器分组。
2.  **打开批量执行功能：** 右键点击该分组，选择“批量执行”。 FinalShell 将会弹出一个新的批量执行窗口。
3.  **配置执行用户：** 在批量执行窗口中，确保“会话用户”选择的是你希望用来执行脚本的用户（通常是 `root` 用户，或者具有 `sudo` 权限的用户）。这将是 FinalShell 登录服务器执行脚本的身份。
4.  **粘贴并执行脚本：**
    *   将你准备好的密码修改脚本粘贴到批量执行窗口的脚本输入框中。
    *   仔细检查脚本内容和目标密码是否正确。
    *   点击“执行”按钮。

![服务器列表与操作演示](/static/images/photo-1507238691740-187a5b1d37b8.jpg)

FinalShell 会逐一连接分组中的每台服务器，并在上面执行你提供的脚本。你可以在批量执行窗口中实时查看每台服务器的执行状态和输出日志。密切关注执行结果，确保每台服务器都显示“成功修改密码”的信息。

## 最佳实践与安全考量

批量修改密码并非一劳永逸，还需要结合其他最佳实践来确保服务器安全。

1.  **使用强密码策略：** 新密码必须足够复杂，避免使用容易猜测的密码。定期强制更换密码，可以显著降低被破解的风险。
2.  **考虑使用 SSH 密钥登录：** 对于更高级别的安全性和便捷性，强烈推荐使用 SSH 密钥对进行身份验证。一旦配置好，你可以禁用密码登录，这会极大提高服务器的安全性，并且管理起来比密码更加方便。你可以参考我们的教程 [FinalShell SSH 密钥登录](/posts/cn-finalshell-ssh-keypair-login/) 来学习如何配置。
3.  **权限最小化原则：** 尽量避免所有操作都使用 `root` 用户。创建具有特定权限的普通用户来执行日常任务，只在必要时使用 `sudo` 提升权限。
4.  **定期审计日志：** 定期检查服务器的认证日志（如 `/var/log/secure` 或 `/var/log/auth.log`），监控异常登录尝试或密码修改记录。
5.  **启用防火墙：** 仅开放必要的端口（如 SSH 默认的 22 端口），限制对 SSH 服务的访问。
6.  **多因素认证 (MFA)：** 如果服务器支持，可以考虑启用 MFA，为登录再加一层保障。

## 常见问题

### Q1：为什么我的脚本执行失败了？

**A：** 脚本执行失败的原因有很多，常见的原因包括：

*   **权限不足：** 执行 `chpasswd` 命令需要 `root` 权限。如果你是以普通用户身份登录 FinalShell，且该用户没有 `sudo` 权限，或者 `sudo` 需要密码，脚本就会失败。请确保 FinalShell 登录的用户是 `root` 或具有免密 `sudo` 权限。
*   **`chpasswd` 命令不存在或禁用：** 某些精简版系统或特殊配置可能没有 `chpasswd` 命令。你可以尝试 `whereis chpasswd` 查看。如果不存在，考虑使用 `passwd --stdin`（但需确认未被禁用）。
*   **网络问题：** 服务器连接不稳定或中断会导致脚本执行失败。请检查 FinalShell 的连接日志。
*   **脚本语法错误：** 仔细检查你的 shell 脚本是否存在语法错误，例如变量名拼写错误、引号不匹配等。
*   **旧密码不正确：** 虽然 `chpasswd` 不需要旧密码，但如果你采用其他方式（如需要交互式输入的 `passwd`），登录 FinalShell 的初始密码必须正确，否则连 SSH 会话都建立不了。

### Q2：我怎么验证密码修改成功了？

**A：** 验证密码修改成功的方法有几种：

*   **FinalShell 批量执行日志：** 查看批量执行窗口中每台服务器的输出。如果脚本成功执行，应该会看到 `用户 root 的密码已成功修改。` 这样的提示。
*   **尝试用新密码登录：** 这是最直接的验证方式。为避免锁住自己，可以先用 FinalShell 打开一个新的会话，尝试用新密码登录其中一台服务器。如果成功，则说明密码修改成功。
*   **尝试用旧密码登录：** 确保旧密码不再有效。
*   **查看 `/etc/shadow` 文件时间戳：** `/etc/shadow` 文件中存储着用户密码的哈希值和相关信息。密码修改后，这个文件的修改时间通常会更新。

### Q3：FinalShell 能不能直接在 GUI 批量修改密码，而不是通过脚本？

**A：** FinalShell 的设计理念是提供强大的终端连接、文件管理和批量任务执行能力。它本身并没有一个直接的“批量修改密码”的图形界面按钮。这主要是因为密码修改是操作系统层面的操作，需要精确的权限控制和非交互式执行，通过脚本来实现是最灵活和可靠的方式。FinalShell 的“批量执行脚本”功能正是为了满足这类需求而设计的。

### Q4：如果我想为每台服务器设置不同的新密码，FinalShell 还能批量操作吗？

**A：** 如果每台服务器需要不同的新密码，那么上面介绍的单一脚本就无法直接满足需求。你需要更复杂的策略：

*   **分批处理：** 根据密码分组服务器，然后针对每个分组执行不同的脚本。
*   **更复杂的脚本：** 编写一个外部脚本（例如 Python），该脚本读取一个包含服务器IP、用户名和新密码的 CSV 文件。脚本会遍历这个文件，然后通过 SSH 库（如 `paramiko`）逐一连接服务器并执行修改密码的命令。这种方法超出了 FinalShell 批量执行脚本的范畴，但 FinalShell 依然可以作为终端或文件传输工具辅助。
*   **配置管理工具：** 对于几十上百台服务器，专业的配置管理工具（如 Ansible、SaltStack、Puppet、Chef）是更合适的选择。它们天生就是为自动化批量管理而生，能更好地处理这种复杂的场景。

### Q5：修改密码后，我的 FinalShell 连接突然超时了，这是怎么回事？

**A：** 如果修改密码后出现连接超时，通常不是密码本身的问题，而是其他相关配置。

*   **SSH 服务重启问题：** 有些系统可能在密码修改后，SSH 服务出现了异常（虽然不常见）。
*   **防火墙策略：** 检查服务器防火墙（如 `firewalld` 或 `ufw`）是否意外修改或阻塞了 22 端口（或其他 SSH 端口）。
*   **SSH 配置变更：** 检查 `/etc/ssh/sshd_config` 是否被意外修改。
*   **网络负载：** 如果服务器负载过高，也可能导致连接超时。

如果你遇到连接超时，可以参考我们的 [FinalShell 连接超时问题修复](/posts/cn-finalshell-connect-timeout-fix/) 这篇文章，里面提供了详细的排查步骤和解决方案。
