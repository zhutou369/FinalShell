*   **Java Runtime Environment (JRE)：**
    *   FinalShell 通常需要 Java 8 或更高版本。如果你不确定是否已安装 Java，可以在命令行中输入 `java -version` 进行检查。
    *   如果没有安装或版本过低，建议前往 [Oracle 官网](https://www.oracle.com/java/technologies/downloads/) 或 [OpenJDK 官网](https://openjdk.org/install/) 下载并安装最新稳定版的 JRE 或 JDK。对于大多数用户，下载 JRE 即可。

### 2. 网络连接状态

确保你的计算机能够正常访问互联网，以便下载 FinalShell 安装包，并且能够连接到你的远程服务器（检查服务器防火墙规则，确保 SSH 端口 22 开放）。

在安装过程中，你可能需要管理员权限来修改系统文件或注册表（Windows）以及将应用拖拽到应用程序目录（macOS），或执行安装脚本（Linux）。

## 首次启动与连接服务器：你的第一步

安装完成后，你就可以启动 FinalShell 并连接到你的第一台远程服务器了。

### 1. FinalShell 界面初探

首次启动 FinalShell，你会看到一个简洁而功能丰富的界面。主要区域包括：

*   **左侧导航栏：** 包含会话列表、SFTP 文件浏览器、性能监控等。
*   **顶部菜单栏：** 提供文件、编辑、工具、视图等选项。
*   **主工作区：** 默认显示一个欢迎页或一个空白会话，未来你打开的 SSH 会话将以标签页的形式显示在这里。

### 2. 添加你的第一个 SSH 连接

1.  **点击“新建”按钮：** 在 FinalShell 界面左上角，通常有一个“新建”按钮或“文件”菜单下的“新建会话”选项。点击它，然后选择“SSH 连接”。
2.  **填写连接信息：**
    *   **名称：** 为你的连接起一个易于识别的名称，例如“我的阿里云服务器”或“生产Web服务器”。
    *   **主机 (IP/域名)：** 填写你的远程服务器的 IP 地址或域名。
    *   **端口：** 默认为 22。如果你的服务器使用了非标准 SSH 端口，请务必修改。
    *   **认证方式：**
        *   **密码：** 选择此项，输入你的 SSH 用户名（如 `root`）和密码。
        *   **公钥：** 推荐的更安全方式。需要你预先在服务器上配置好公钥，并在这里指定你的私钥文件路径。
    *   **用户名：** 远程服务器的登录用户名。
    *   **密码：** 远程服务器的登录密码。
3.  **保存并连接：** 填写完毕后，点击“保存”按钮。然后，你可以在左侧的会话列表中找到新添加的连接，双击它即可尝试连接。

### 3. 公钥认证：更安全的连接方式

公钥认证比密码认证更加安全，因为它不通过网络传输实际的密码。

1.  **生成 SSH 密钥对：**
    *   如果你还没有 SSH 密钥对，可以在本地计算机的终端（或 Git Bash on Windows）中运行 `ssh-keygen -t rsa -b 4096` 命令生成。按照提示操作，通常会在 `~/.ssh/` 目录下生成 `id_rsa` (私钥) 和 `id_rsa.pub` (公钥) 文件。
2.  **上传公钥到服务器：**
    *   将 `id_rsa.pub` 文件中的内容复制到远程服务器的 `~/.ssh/authorized_keys` 文件中。
        *   你可以先用密码方式登录服务器，然后创建或编辑该文件：
            ```bash
            mkdir -p ~/.ssh
            chmod 700 ~/.ssh
            vi ~/.ssh/authorized_keys # 将你的公钥粘贴进去
            chmod 600 ~/.ssh/authorized_keys
            ```
3.  **配置 FinalShell 使用私钥：**
    *   在 FinalShell 的“新建 SSH 连接”窗口中，选择“认证方式”为“公钥”。
    *   点击“私钥文件”旁边的文件夹图标，导航到你本地计算机的 `~/.ssh/id_rsa` (或你自定义的私钥文件) 路径并选择。
    *   输入私钥的密码（如果你的私钥设置了密码）。
    *   点击“保存”并连接。

*   **远程端口转发：** 将远程服务器上的端口流量转发到本地或另一个网络。
*   **动态端口转发 (SOCKS 代理)：** 提供一个 SOCKS 代理服务，所有通过该代理的流量都会通过 SSH 隧道转发。

## 常见问题排查与解决

在使用 FinalShell 过程中，你可能会遇到一些常见问题。

*   **连接失败：Connection refused (连接被拒绝)**
    *   **原因：** 服务器的 SSH 服务未运行、服务器防火墙阻止了 SSH 连接、SSH 端口错误。
    *   **解决：** 确保服务器 SSH 服务 `sshd` 正在运行 (`systemctl status sshd`)；检查服务器安全组或防火墙规则，确保 SSH 端口 (默认为 22) 对你的 IP 地址开放；确认 FinalShell 中填写的端口号是否正确。
*   **认证失败：Authentication failed (认证失败)**
    *   **原因：** 用户名或密码错误、私钥文件不正确、私钥权限问题、服务器端公钥配置错误。
    *   **解决：** 仔细检查用户名和密码；如果使用公钥认证，确保私钥文件路径正确，私钥没有被损坏，并且服务器上的 `~/.ssh/authorized_keys` 文件内容正确且权限 (`chmod 600`) 正确。
*   **“Java not found” 或无法启动**
    *   **原因：** 未安装 Java 运行时环境（JRE）或版本不兼容，环境变量未配置正确。
    *   **解决：** 确保你的系统已安装 Java 8 或更高版本。在 Windows 上，确认 Java 的 `bin` 目录已添加到 PATH 环境变量。在 Linux 上，可以使用 `update-alternatives` 等工具设置默认 Java 版本。
*   **界面卡顿或反应迟钝**
    *   **原因：** 可能内存不足，或者正在显示大量日志输出导致渲染压力大。
    *   **解决：** 尝试关闭其他占用内存的应用程序；检查服务器是否有大量实时日志输出到终端；尝试重启 FinalShell。

## SSH 安全最佳实践

安全永远是远程连接服务器的首要考量。遵循以下最佳实践，可以显著提升你的 SSH 连接安全性。

1.  **使用强密码：** 为你的 SSH 账户设置长且复杂的密码，结合大小写字母、数字和特殊字符。
2.  **优先使用公钥认证：** 公钥认证比密码认证更安全，因为它不通过网络传输实际的密码，并且私钥始终保留在本地。
3.  **禁用密码认证（如果可能）：** 一旦成功配置了公钥认证，考虑在服务器端禁用密码认证，以防止暴力破解攻击。
    *   编辑 `/etc/ssh/sshd_config` 文件，将 `PasswordAuthentication yes` 改为 `no`，然后重启 SSH 服务。
4.  **限制 root 用户直接登录：** 尽量避免直接使用 `root` 账户通过 SSH 登录。创建一个普通用户，并赋予其 `sudo` 权限，需要时再切换到 root。
    *   编辑 `/etc/ssh/sshd_config` 文件，将 `PermitRootLogin yes` 改为 `no`。
5.  **更改默认 SSH 端口：** 将默认的 22 端口更改为其他非常规端口，可以减少自动化扫描攻击的频率。
    *   编辑 `/etc/ssh/sshd_config` 文件，将 `Port 22` 改为其他端口，例如 `Port 2222`。**注意：** 更改后，务必更新服务器防火墙规则，并确保 FinalShell 中也使用新端口连接。
6.  **定期更新软件：** 保持你的操作系统、SSH 客户端 (FinalShell) 和服务器端的 SSH 服务 (OpenSSH) 更新到最新版本，以获取最新的安全补丁。
7.  **配置防火墙：** 在服务器上启用防火墙（如 `ufw` 或 `firewalld`），并只允许信任的 IP 地址访问 SSH 端口。
